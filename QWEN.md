# Telegram Bot Mute — Контекст для Qwen

## Обзор проекта

**Telegram Bot Mute** — это бот для автоматической модерации новых участников в Telegram-группах. При вступлении нового пользователя бот автоматически ограничивает его права на отправку сообщений ("мутит") до тех пор, пока пользователь не свяжется с администратором.

### Основные функции

- **Авто-мут новых участников**: Запрет на отправку сообщений, медиа и ссылок
- **Приветственное сообщение**: Отправка инструкции с кнопкой связи с администратором
- **Авто-очистка**: Удаление приветственного сообщения через 30 секунд
- **White-list**: Ограничение работы бота конкретными чатами через `ALLOWED_CHATS`
- **Ротация логов**: Файл `bot.log` с максимальным размером 5 МБ и 3 резервными копиями
- **Защита памяти**: Автоматическая очистка `PROCESSED_USERS` через 10 минут

## Настройка и запуск

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Настройка переменных окружения

Создайте файл `.env` в корне проекта:

```ini
BOT_TOKEN=ваш_токен_от_botfather
```

### 3. Конфигурация в коде (`Bot_mute.py`)

| Параметр | Описание | Значение по умолчанию |
|----------|----------|----------------------|
| `MY_ID` | Telegram ID владельца бота | `362752154` |
| `ALLOWED_CHATS` | Список ID разрешённых чатов | `[]` (все чаты) |
| `ADMIN_CONTACT_URL` | Ссылка на администратора для кнопки | `"https://t.me/frau_ponomareva"` |
| `WELCOME_MESSAGE_TEXT` | Шаблон приветствия (тег `{full_name}`) | `"Привет, {full_name}! Чтобы писать в чат, свяжись с администратором."` |

### 4. Запуск бота

```bash
python Bot_mute.py
```

## Команды бота

| Команда | Описание | Доступ |
|---------|----------|--------|
| `/start` | Включает мут новых участников в текущем чате | Администраторы группы |
| `/stop` | Отключает мут новых участников в текущем чате | Администраторы группы |
| `/log` | Отправляет последние 100 строк логов в ЛС владельцу | Только `MY_ID` |

## Архитектура кода

### Основные компоненты

- **`BOT_RUNNING`** — словарь `{chat_id: bool}` для отслеживания состояния бота в чатах
- **`PROCESSED_USERS`** — множество ключей `"{chat_id}:{user_id}"` для предотвращения дублирования
- **`escape_markdown()`** — экранирование специальных символов для MarkdownV2

### Обработчики событий

| Обработчик | Событие | Описание |
|------------|---------|----------|
| `start` | `NEW_CHAT_MEMBERS` | Обработка новых участников из сообщения |
| `chat_member_update` | `ChatMemberHandler` | Обработка изменений статуса участников |
| `start_bot` | `/start` | Включение бота в чате |
| `stop_bot` | `/stop` | Выключение бота в чате |
| `send_log` | `/log` | Отправка логов владельцу |

### JobQueue задачи

- **`delete_message`** — удаление приветственного сообщения через 30 секунд
- **`remove_from_processed`** — очистка `PROCESSED_USERS` через 10 минут

## Логирование

Логи записываются в `bot.log` с использованием `RotatingFileHandler`:

- Максимальный размер файла: 5 МБ
- Количество резервных копий: 3
- Кодировка: UTF-8
- Уровень логирования: INFO

## Практики разработки

### Стиль кода

- Типизация: используются аннотации типов (`Dict`, `Any`, `List`)
- Документирование: каждая функция имеет docstring
- Обработка ошибок: try/except блоки с логированием исключений
- Константы: настройки вынесены в начало файла

### Безопасность

- Проверка прав администратора через `check_admin_permissions()`
- White-list чатов через `ALLOWED_CHATS`
- Проверка владельца бота для команды `/log`
- Пропуск ботов при обработке новых участников

### Оптимизация

- Чтение последних 100 строк лога через `deque` без загрузки всего файла
- Использование `PROCESSED_USERS` для предотвращения дублирования действий
- Автоматическая очистка памяти через таймеры JobQueue

## Известные ограничения

- JobQueue требует установленную библиотеку `apscheduler` (входит в `python-telegram-bot[job-queue]`)
- При отсутствии JobQueue авто-удаление сообщений и очистка `PROCESSED_USERS` не работают
- Бот требует права администратора для ограничения участников
